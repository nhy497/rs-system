## 文件說明

### 合併記錄格式

每個合併記錄應包含：

1. **執行摘要** - 簡述合併目的和結果
2. **問題描述** - 說明原始 PR 的問題背景
3. **解決方案** - 描述採用的解決方案
4. **為何手動合併** - 解釋為何不能直接合併
5. **核心改動** - 列出所有改動的檔案
6. **改動統計** - 原 PR 與手動合併的比較
7. **結論** - 總結成果與經驗

### 執行報告格式

`EXECUTION_REPORT.md` 記錄了實際執行步驟：

1. **執行摘要** - 任務狀態和改動量對比
2. **已完成任務** - 詳細的任務清單
3. **提交的改動** - Git 提交記錄
4. **最終狀態** - 分支狀態
5. **經驗總結** - 關鍵學習點

## 命名規則

合併記錄檔名格式：`YYYYMMDD_PR-X-Y-合併記錄.md`

- `YYYYMMDD` - 執行日期
- `PR-X-Y` - 相關 PR 編號（可多個）
- 使用繁體中文描述

## 何時需要手動合併

當 PR 出現以下情況時，建議手動合併：

1. **大量不必要檔案** - 如 node_modules/, dist/ 等
2. **倉庫体積爆炸** - 改動量超過百萬行
3. **Git 歷史污染** - 大量無意義的改動
4. **包含有價值改動** - PR 中混有真正需要的代碼
