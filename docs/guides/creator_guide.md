# 創作者帳戶使用指南

## 🎯 創作者帳戶特性

創作者帳戶是專為系統管理員設計的特殊帳戶，具有以下特性：

### ✅ 可執行的操作

1. **查看所有用戶資料**
   - 在「用戶管理」頁面查看所有註冊用戶
   - 查看每個用戶的課程記錄數量
   - 點擊「查看課程」按鈕查看用戶的詳細課程內容

2. **查看課程詳情**
   - 查看任何用戶的課程記錄
   - 查看課程的完整資訊（日期、學生、技能進度等）
   - 無法修改或刪除其他用戶的課程

3. **管理用戶帳戶**
   - 刪除用戶帳戶（除了當前登入的帳戶）
   - 查看系統統計資訊

### ❌ 不可執行的操作

1. **不能新增課程記錄**
   - 儲存按鈕會被禁用
   - 嘗試儲存時會顯示錯誤訊息
   - 創作者帳戶僅限查看，不能創建新課程

2. **不能修改課程記錄**
   - 無法編輯其他用戶的課程資料
   - 無法刪除課程記錄

## 🔑 登入資訊

- **用戶名稱**: `creator`
- **密碼**: `1234`

## 📋 使用流程

### 查看用戶課程記錄

1. 使用創作者帳戶登入系統
2. 點擊左側菜單的「用戶管理」
3. 在用戶列表中找到目標用戶
4. 點擊「📋 查看課程」按鈕
5. 在彈出視窗中瀏覽該用戶的所有課程記錄
6. 點擊「查看詳情」查看單一課程的完整資訊

### 管理用戶帳戶

1. 進入「用戶管理」頁面
2. 查看系統統計：
   - 總用戶數
   - Creator 數量
   - 普通用戶數量
3. 刪除用戶（如需要）：
   - 點擊用戶旁的「刪除」按鈕
   - 確認刪除操作

## 🎨 界面提示

當以創作者身份登入時，您會看到：

1. **側邊欄用戶信息**：顯示「👑 Creator」標籤
2. **課堂概覽頁面**：頂部顯示黃色提示框
   - 說明創作者模式的限制
   - 提示前往「用戶管理」查看其他用戶課程
3. **儲存按鈕**：顯示「🔒 Creator 不能新增記錄」並被禁用

## 🔍 技術細節

### 數據結構

每個課程記錄包含以下欄位：
```javascript
{
  id: "checkpoint_${timestamp}_${random}",
  userId: "user_${timestamp}_${random}",
  classDate: "2026-01-26",
  className: "P3A",
  // ... 其他課程資料
  createdAt: "2026-01-26T10:00:00.000Z",
  updatedAt: "2026-01-26T10:00:00.000Z"
}
```

### 權限檢查

系統在以下位置進行權限檢查：

1. **保存課程記錄時** (`btnSave` 點擊事件)
2. **STORAGE_MANAGER.saveCheckpoints()** 方法
3. **頁面初始化時** (`updateUserInfo()` 函數)

## 📝 常見問題

### Q: 為什麼創作者看不到其他用戶的課程？

A: 請確保：
1. 其他用戶已經建立課程記錄
2. 課程記錄包含 `userId` 欄位
3. 在「用戶管理」頁面查看，而不是在「課堂概覽」頁面

### Q: 創作者可以編輯課程記錄嗎？

A: 不可以。創作者帳戶僅限查看，不能新增、修改或刪除課程記錄。

### Q: 如何測試創作者功能？

A: 
1. 創建一個普通用戶帳戶
2. 登入並建立幾筆課程記錄
3. 登出，然後用創作者帳戶登入
4. 前往「用戶管理」查看該用戶的課程

## 🔧 開發者備註

修改的核心檔案：
- `system.js` - 添加創作者權限檢查和用戶課程查看功能
- 關鍵函數：
  - `viewUserCheckpoints()` - 查看用戶課程記錄
  - `viewCheckpointDetail()` - 查看單一課程詳情
  - `updateUserInfo()` - 禁用創作者的新增功能
  - `STORAGE_MANAGER.saveCheckpoints()` - 權限檢查
